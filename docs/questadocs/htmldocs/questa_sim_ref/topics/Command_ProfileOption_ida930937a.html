profile option

Changes how profiling data are reported.

Syntax

profile option collapse_sections [on | off | status]

profile option collect_calltrees [on | off | status]

profile option solveprofile_enable [on | off | status]

profile option solveprofile_details [on | off | status]

profile option solveprofile_testgen [on | off | status]

Description

Use the “profile option" command to view or modify parameters that affect the behavior of the "profile" command. Invoking the "profile option <parameter>" without an additional argument will display the current status of that parameter.

Parameters

collapse_sections [on | off | status]

Groups profiling data by section. A section consists of regions of code such as VHDL processes, functions, or Verilog always blocks. By default all profiling data are reported on a per line basis.

  • on — (optional) Enables profiling of data by section.

  • off — (optional) Disables profiling of data by section. (default)

  • status — (optional) Returns the current setting of the parameter.

collect_calltrees [on | off | status]

Collects data for call trees, showing which functions or routines call which others. By default this information is not collected. Simulation time and resource usage will increase if you enable collection of this data.

  • on — (optional) Enables call tree collection.

  • off — (optional) Disables call tree collection. (default)

  • status — (optional) Returns the current setting of the parameter.

solveprofile_enable [on | off | status]

Enables/disables constraint solver profiling (default is on). This option allows you to disable constraint solver profiling when only statistical performance profiling is desired.

  • on — (optional) Enables profiling of calls to constraint solver (default).

  • off — (optional) Disables profiling of calls to constraint solver.

  • status — (optional) Returns the current setting of the parameter.

solveprofile_details [on | off | status]

Enables reporting of details of the randsets being computed (default is off).

  • on — (optional) Enables detailed constraint solver profile reporting.

  • off — (optional) Disables detailed constraint solver profile reporting (default).

  • status — (optional) Returns the current setting of the parameter.

solveprofile_testgen [on | off | status]

Enables creation of testcases for the randsets that have been computed (default is off). This option requires solveprofile_details to be on to take effect.

  • on — (optional) Enables testcase creation for randsets.

  • off — (optional) Disables testcase creation for randsets (default).

  • status — (optional) Returns the current setting of the parameter.

Examples

  • Enable profiling of collapsed processes and functions.

    profile option collapse_sections on

    Returns:

    # Profiling will now report collapsed processes and functions

  • Turn off reporting of collapsed processes and functions.

       profile option collapse_sections off

    Returns:

    # Profiling will now NOT report collapsed processes and functions
