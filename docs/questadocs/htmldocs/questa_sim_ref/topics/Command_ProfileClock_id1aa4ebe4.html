profile clock

Changes the source of the timer for the profiler. This command is only applicable in Linux environments. In Windows environments, the profile timer remains the default wall clock regardless of which argument you specify.

Syntax

profile clock [ default | cpu | legacyCpu | monotonicWall | status ]

Arguments

default

Sets the timer source to be the wall clock.

cpu

Sets the timer source to be the CPU clock. Only times while the process is consuming cpu time.

wall

Sets the timer source to be the wall clock.

legacyCpu

Only use this argument for special situations when the wall clock will not work, and know that the process only has one such timer. If PLI code is using this clock for licensing heartbeat intervals then the code will fail. This option also results in a much larger latency between the timer going off and when the signal is called.

monotonicWall

Sets the timer source to be similar to the wall timer except that Linux disallows the value from ever being smaller than the previous value.

status

Returns the current setting of the profile clock.

Description

This command attempts to provide workaround options for occasions when running profiling on a simulation results in failure. This is usually due to improper system calls in the source code, so use caution if attempting to resolve the issue with profile clock.

Examples

profile clock cpu
