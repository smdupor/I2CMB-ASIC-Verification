Interactive Mode

Use this mode to launch interactive simulations from VRM.

Invocation Procedure

  1. From the command line, add -interactive to the vrun command.

  2. From the GUI, check the Run simulations interactively check box on the Runtime tab of the VRM Configuration dialog box

  3. Invoke the simulation using any of the following methods:

    • VRM Cockpit: Run or Run Special menu items.

    • VRM Results Window: Run Again or Run Selected menu items.

    • Results Analysis Window: Re-run in Run Manager.

Interactive Mode Behavior

The behavior of vrun during an interactive mode regression depends on the launch type (launch attribute) of the Action element in your RMDB file.

  • If you set launch to vsim, vrun expects TCL commands.

  • If you set launch to exec, vrun expects commands for the specific interpreter, which is typically your default operating system shell.

  • If you do not specify launch, the default is vsim.

Interactive Mode in either vsim or exec launch Mode

A regression in interactive mode disables execution timeouts, but honors and checks queuing timeouts.

Upon completion of an Action in interactive mode, VRM posts a done event to the status event log with the status of passed and no extended status. VRM also suppresses all pass/fail analysis and post-execution steps such as auto-merge, auto-triage, and auto-delete including the following functions:

  • It does not check the UCDB TESTSTATUS attribute, if any.

  • It does not call the AnalyzePassFail user-definable procedure.

  • The Action always returns a passed status, even if errors occurred.

  • The immediate rerun functionality is disabled.

  • The Action does not return any errors or warnings to the vrun output (except for queuing timeouts).

  • It does not update the saved seed values.

Example Scenario

Assume you just ran a regression with six runnables and everything completed, but one had an error. Now you want to run just that runnable and have the GUI open for you to work with manually. Also assume that there was some sort of stop on error scenario and you want to debug the simulation.

Note:

If you run interactive mode on your full regression, VRM will not stop you, but it will then go through and open an interactive window for every runnable.

Prerequisite

Create a Run Configuration to use for interactive mode.

  1. Open the VRM GUI

  2. Select VRM Cockpit > Add Item > Run Configuration

    This displays the Create VRM Configuration dialog box.

  3. In the General tab, set VRM Configuration Name to Interactive Mode.

  4. In the Runtime tab, set Run Mode to Run simulations interactively.

  5. In all other tabs, set the values as you require for your environment.

  6. OK

Procedure

  1. Run your regression, for example:

    vrun -gui nightly

    and assume that a runnable failed.

  2. In the VRM Results window, right-click on your failing runnable and select Run Selected > Interactive mode

    where Interactive Mode is the name of the VRM Configuration you created.

    This opens the Edit Regression Options dialog box.

  3. Alter the Edit Regression Options dialog box as needed and click Run.

    This opens a Questa SIM GUI on your desktop, which allows you to directly interact with your simulation

  4. Debug your simulation

  5. When you have completed your debug, you can allow your simulation to finish normally or end the simulation with a quit -f command.

  6. When the simulation is complete and the GUI closed, your vrun regression will continue on with its processes.
