Generate a Project File by Invoking an Existing Batch-mode vrun in GUI Mode

As mentioned above, any time a batch-mode regression is launched at the same time that a GUI session is launched, a configuration corresponding to the command-line options used to launch the batch-mode regression is added to the VRM Cockpit.

If the -project command-line option was also used to open a valid project file, then this new configuration will automatically be saved to the project file. If no project file is loaded, then the user may use the File > Vrun Output main-window menu item to save the configuration (currently in the VRM Cockpit) to a new project file.
