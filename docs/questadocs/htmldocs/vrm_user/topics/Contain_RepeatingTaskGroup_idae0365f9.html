Repeating a Task or a Group

In some cases, such as when constrained random techniques are used to generate stimulus for the device under test (DUT), it can be useful to be able to repeat a single Task (that is, simulation) or Group of Tasks multiple times as part of the same regression run.

In the RMDB database itself, there are three ways you can cause a single runnable in the RMDB to be expanded into multiple runnable instances in a regression.

  • Add a repeat attribute to the Runnable representing the Task or Group that is to be executed multiple times. The value of this attribute determines the number of times the Task or Group repeats. If the value of the repeat attribute is non-numeric, or if it is set to a number less than one, the effective value is assumed to be one and the Runnable will not repeat.

  • Use the foreach attribute in the runnable tag.

  • Use the testlist attribute to cause vrun to read a testlist file where you can have multiple instances of the same runnable in a regression run.
