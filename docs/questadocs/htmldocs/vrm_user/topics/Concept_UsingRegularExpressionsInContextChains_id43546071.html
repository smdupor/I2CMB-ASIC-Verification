Using Regular Expressions in Context Chains

While the globbing characters provide a great deal of flexibility in selecting multiple Runnables with one context chain, even more selection flexibility can be obtained by adding the -regex option to the vrun command line.

The -regex option enables the use of regular expressions (as defined in the Tcl regexp command) instead of simple globbing. For example, the globbed command in the “Globbing in Context Chains” might also have been written as follows:

% vrun -regex -show -include '/n.*'
nightly/directedtests/...
nightly/randomtests/...
nightly/unittests/...

Note that instead of n*, you write n.* meaning the letter “n” followed by any number of characters. When the -regex option is specified, all components of all context chains listed on the command line are matched using regular expressions (as opposed to glob expressions). Other than that, processing is identical in the two modes.

The effect of the -regex option is global for any given run. Globbed context chains cannot be mixed with regular expression context chains in the same command.
