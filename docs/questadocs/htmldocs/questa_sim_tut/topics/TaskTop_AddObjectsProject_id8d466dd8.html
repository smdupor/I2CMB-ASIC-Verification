Add Objects to the Project

Once you click OK to accept the new project settings, a blank Project window and the “Add items to the Project” dialog box will appear.

From the dialog box (Figure 1) you can create a new design file, add an existing file, add a folder for organization purposes, or create a simulation configuration (discussed below).

Figure 1. Adding New Items to a Project

Procedure

Add two existing files.
  1. Click Add Existing File.

    This opens the Add file to Project dialog box (Figure 2). This dialog box lets you browse to find files, specify the file type, specify a folder to which the file will be added, and identify whether to leave the file in its current location or to copy it to the project directory.

    Figure 2. Add file to Project Dialog Box

  2. Click the Browse button for the File Name field. This opens the “Select files to add to project” dialog box and displays the contents of the current directory.
  3. Verilog: Select counter.v and tcounter.v and click Open or VHDL: Select counter.vhd andtcounter.vhd and click Open.

    This closes the “Select files to add to project” dialog box and displays the selected files in the “Add file to Project” dialog box (Figure 2).

  4. Click OK to add the files to the project.
  5. Click Close to dismiss the Add items to the Project dialog box.

    You should now see two files listed in the Project window (Figure 3). Question-mark icons in the Status column indicate that the file has not been compiled or that the source file has changed since the last successful compile. The other columns identify file type (for example, Verilog or VHDL), compilation order, and modified date.

    Figure 3. Newly Added Project Files Display a ’?’ for Status
