Editing Transform Rules Files in the GUI

Transform rules files can be created/edited within the GUI, through the Transform Rules File Editor dialog box.

Procedure

  1. Open the Create Results Analysis Database dialog box by choosing:

    File > New > Results Analysis Database...

  2. Select the Xform tab.
  3. To open a new Transform Rules File
    1. Click New
    2. Enter a name in the File name field and click Save.
  4. To edit an existing Transform Rules File
    1. Select a transform rules file (.xform) file from the list that appears
    2. Click Open.
  5. Step 3 or 4 opens the Transform Editor dialog box, shown in Figure 2. You can use this editor to create, view, edit and apply transform rules to a Results Analysis database.
    Figure 1. Transform Rule File Editor

  6. Opening an existing .wlf file and creating rules from messages therein is one of the easiest ways to create new rules:
  7. Click the folder icon in the right top corner of the window.
  8. Open either a .log file or .wlf file from the list of files.
  9. Select a message to use as a template for a new rule:
    1. Click to select a message.
    2. Right click and choose Construct New Rule from Selection.

    A tab entitled “NewRule1” tab appears in the window, containing the selected message and a regular expression based on that message, as shown in Figure 2.

    Figure 2. Editing New Transform Rule from a Message
