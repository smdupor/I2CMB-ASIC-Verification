How Links are Established

The links are performed automatically when you merge an imported testplan with actual tests you have run. Key to this linking, however, is the correct association of the design constructs — various linkable constructs in the design — to the verification plan items.

The link between a coverage item and a testplan section is accomplished, essentially, before the plan is imported, using one of the following methods:

  • For spreadsheets — by entering the coverage item name into the “Link” column/field, and a coverage construct keyword into the “Type” column/field (see Figure 1). Table 1 is a complete list of the linkable coverage constructs for columns/fields.

  • For word-processing documents — by placing the coverage item name using the “Link:” label, and placing one of the listed design constructs into using the “Type:” label. Table 1 lists the names of all labels recognized by the linking mechanism.
