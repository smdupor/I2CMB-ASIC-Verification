The Capacity Window

The Capacity-Object and Capacity-Line windows show data about current memory allocation. You can use them to find areas where memory usage is causing problems, and to display count, memory usage, and other information.

The default view shows course grain analysis totals for two object types:

  • Static objects — analysis includes the object count and current memory usage.

  • Dynamic objects — analysis includes object count, current memory usage, peak memory usage, and the time peak memory usage occurred.

Questa SIM collects data as either a coarse-grain analysis (default) or a fine-grain analysis of memory capacity. The main difference between the two levels is the amount of capacity data collected. You must enable fine-grain analysis to view count, current and peak memory allocation for each class type, aggregate information about the class type, including the current filename and line number where the allocation occurred. Refer to Levels of Capacity Analysis for more information.

Refer to Capacity-Object and Capacity-Line Windows in the GUI Reference Manual, and Capacity Analysis in this manual for more information about viewing class memory usage. You can also display capacity data in the Wave Window. Refer to Displaying Capacity Data in the Wave Window for more information.
