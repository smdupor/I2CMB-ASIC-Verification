Understanding Communication Semantics

A Questa Verification IP uses a number of standard communications semantics that define the mode of transmission, or observation of, a transaction across the protocol interface. A Questa Verification IP transaction type is defined as an MVC_transaction, an MVC_message or an MVC_stripe. Each transaction type supports a sub-set of the available communications semantics:

  • An MVC_Transaction is bi-directional and can be transmitted using the Activate, Activating or Activates semantics.

  • An MVC_Message or MVC_Stripe is uni-directional can be transmitted using the Send, Sending or Sent semantics.

  • Any MVC_Transaction, MVC_Message or MVC_Stripe can be observed using the Receive, Receiving or Received semantics.
