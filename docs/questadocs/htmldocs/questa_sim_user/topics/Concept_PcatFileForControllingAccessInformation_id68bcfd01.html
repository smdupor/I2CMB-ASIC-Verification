PCAT File for Controlling Access Information

There are two flows for using a PCAT file in your PLI environment, where you specify only access information.

It is suggested that you use a PCAT file to specify design object access requirements, as opposed to using the +acc option to the vopt command). The primary advantage to using PCAT files is that it is easy to encapsulate design object access requirements for each PLI systf. This encapsulation allows the simulator to optimize performance in the presence of PLI without the need for a PLI Learn flow.

  • For re-usable PLI-based IP's:

    • Compile and distribute in .so form.

    • Specify the .so file using the -pli argument to vsim.

    • Define (*vlog_startup_routines[]) with function linkage in the C code.

    • Distribute a PCAT file that contains access information, but no function linkage.

  • For home-grown PLI C code:

    • Use the PLI Autocompile flow.

    • Define (*vlog_startup_routines[]) with function linkage in the C code.

    • Create a PCAT file that contains access information, but no function linkage.
