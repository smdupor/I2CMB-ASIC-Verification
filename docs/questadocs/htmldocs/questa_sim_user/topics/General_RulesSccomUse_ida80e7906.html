Rules for sccom Use

The rules governing when and how you must use sccom are as follows.

  • You should compile all SystemC code with sccom.

  • You must compile using sccom for the top-level, for mixed-language access (anywhere there is a language boundary) or for debugging (waveform logging, Tcl command interaction).

  • When using sccom, you should not use the -l compiler option to point the compiler at any search directories containing OSCI or any other vendor supplied SystemC header files. sccom does this for you accurately and automatically.

  • If you do use the standalone compiler to compile C/C++ functionality into archives or shared objects, you must then link your design using the -L and -l options with sccom ‑link. These options effectively pull the non-SystemC C/C++ code into a simulation image that is used at runtime.

Failure to follow the above rules can result in link-time or elaboration-time errors due to mismatches between the OSCI or any other vendor supplied SystemC header files.
