Setting a Breakpoint on a Function Entry

The bp in command accepts function and task names to support a “stop in function entrance” capability. This allows you to set a breakpoint without needing the precise file and line number of a given function. The bp command places a breakpoint on the first executable line of the specified task or function.

Procedure

Enter the bp in command with a function or task name.

bp in <function_name>

Examples

bp in /uvm_pkg::uvm_component::set_config_int -cond {this == @myunit@1}

bp in /mem_agent_pkg::mem_seq5::body
