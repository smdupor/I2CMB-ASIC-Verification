OVM Hierarchy Window

Display this window by choosing the View > OVM > OVM Hierarchy menu item.

The OVM Hierarchy window contains hierarchical information about your OVM environment.

  • Name — Provides a hierarchical view of the OVM classes used. The names are based on the class names created in your OVM environment.

  • Type — Identifies the type of object listed in the Name column. Examples include: component, sequencer, ovm_port, tlm, fifo, sequencer, sequence.

  • Phase — Identifies whether the object in the Name column has been started or completed.
