Adding User Entries to the RNG Trace File

To help identify/correlate Verilog code with related RNG Trace activity, you can add annotations to the RNG trace.

Procedure

Use the $rngtrace_message(string) system task to write an entry to the RNG trace.

A standard trace entry is written, with the <operation> of the entry supplied by the argument (string) to $rngtrace_message. The <type> of the entry is "p*".

Examples

For the Verilog code:

$rngtrace_message("Prior to get_randstate");
rs = process::self().get_randstate();
c1 = new();
process::self().set_randstate(rs);
$rngtrace_mesage("After reset randstate");

The corresponding RNG trace entries look like the following:

<rng>p* /top/#INITIAL#1767962654#: -> MS547527bb5f9e2c20661dwea9091ed841d, Prior to get_randstate @ test.sv(66)<rng>p+ /top/#INITIAL#1767962654#: MS547527bb5f9e2c20661dwea9091ed841d -> MSf9e2c20661d2ea90976b2d65b8bdb376, (get class seed, seed:3948585912) @ test.sv (68)<rng>c= /top/#INITIAL#1767962654#: -> MSd1ef12ef321778f006bc6d297504e6e2, (class1::class initialize, seed:3948585912) @ tesMSf9e2c20661d2ea90976b2d65b8bdb376t.sv (68)<rng>p= /top/#INITIAL#1767962654#: MSf9e2c20661d2ea90976b2d65b8bdb376 -> MS547527bb5f9e2c20661dwea9091ed841d, (process::set_randstate) @ test.sv(69)<rng>p* /top/#INITIAL#1767962654#: -> MS547527bb5f9e2c20661dwea9091ed841d, After retset randstate test.sv (70)
Note:

Since get_randstate() does not modify the randstate, it is not reported by RNG trace.
