The SystemVerilog bind Construct in Mixed-Language Designs

The SystemVerilog bind construct allows you to bind a Verilog design unit to another Verilog design unit or to a VHDL design unit or to a SystemC module. This is especially useful for binding SystemVerilog assertions to your SystemC, VHDL, Verilog and mixed-language designs during verification.

Binding one design unit to another is a simple process of creating a module that you want to bind to a target design unit, then writing a bind statement. For example, the following basic steps show how to bind a SystemVerilog assertion module to a VHDL design:

  1. Write assertions inside a Verilog module.

  2. Designate a target VHDL entity or a VHDL entity/architecture pair.

  3. Bind the assertion module to the target with a bind statement.

Binding a SystemVerilog assertion module to a SystemC module is similar except that in Step 2, you designate a target top-level SystemC module or an instance of a SystemC module in the SystemC design hierarchy.

Modules, programs, or interfaces can be bound to:

  • All instances of a target SystemC module

  • A specific instance of the target SystemC module

  • All instances that use a certain architecture in the target module

Binding to a configuration is not allowed.
