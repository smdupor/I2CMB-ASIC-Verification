Auto Exclusions

Exclusions are automatically applied to certain code constructs. An example is assertion code, which is normally not considered part of the “design”. It is not normally desired to have assertions participate in code coverage.

Another example is FSM state exclusions. By default, excluding a state excludes all transitions to and from that state. To explicitly control FSM auto exclusions set the vsim argument -autoexclusionsdisable using a command such as:

vsim <your vsim args> –autoexclusionsdisable=fsm

To change the default behavior of the tool, set the variable AutoExclusionsDisable in the modelsim.ini file.

Auto Exclusions in the Source Window

The Source window and coverage reports display the coverage data with special indications on lines that contain auto exclusions.

Refer to “Coverage Data in the Source Window” in the GUI Reference Manual for a full list of icons (such as EA), and to Table 1 for a list of codes (such as “EBCS”) that appear in the coverage report and are associated with auto exclusions.
