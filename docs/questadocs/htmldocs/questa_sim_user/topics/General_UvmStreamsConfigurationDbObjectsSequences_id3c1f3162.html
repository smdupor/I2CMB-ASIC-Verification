UVM Streams, Configuration DB Objects, and Sequences

The UVM Details window displays information about UVM objects you select in the Structure window.

Figure 1 shows an example of Stream mode, which displays any existing transaction streams that exist at or below the UVM component hierarchy you selected in the Structure window. You can drag UVM stream objects into the Wave window.

Figure 1. UVM Details Window Stream Mode

Figure 2 shows an example of ConfigDB mode, which displays any configuration database items that are accessible to the UVM component you selected in the Structure window. It displays the value, the type, and whether the configuration item is written to, or read. All of the configuration database items are listed when you select the uvm_root component. You can use this window to help debug testbench name mismatches which can cause failure of the testbench.

Figure 2. UVM Details Window ConfigDB Mode

Figure 3 shows an example of the Processes window displaying hierarchy mode, includeing a list of sequencers and the active sequences running under them for the currently selected UVM hierarchy.

Figure 3. UVM Details Window Sequence Mode
