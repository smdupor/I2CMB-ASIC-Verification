Commands Supporting Negative Timing Check Limits

By default, the simulator supports negative timing check limits in Verilog $setuphold and $recrem system tasks.

Using the +no_neg_tchk argument with the vsim command causes all negative timing check limits to be set to zero.

Models that support negative timing check limits must be written properly to be evaluated correctly. These timing checks specify delayed versions of the input ports. The delayed versions are used for functional evaluation. The following sections describe the correct syntax for $setuphold and $recrem.
