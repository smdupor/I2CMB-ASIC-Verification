Options Used for Building the Merge List

The vcover parallelmerge arguments used to build the merge list are the following.
Table 1. Merge List Options

Argument

Function Performed

Setting

-genlist

Generates file list from $cwd or $cwd/CoverStore database location

Default is the current directory

-genlistfrom

Generates the file list from given database locations

NA

-filelist

Specifies the file list name and path

Default is $cwd/parallellist

-ucdbname

Defines the name pattern list of UCDB file(s) used to search and build the merge list

Default is the string “*.ucdb”

-teststatus

Defines the file list for merging based on the TESTSTATUS value in the test record of each UCDB

Options:

  • all — all tests in the original file list are added to the merge file list

  • passed — only the TESTSTATUS results of Note and Warning are added

  • failed — only TESTSTATUS results of Error and Fatal are added

Note: If the leaf file is a merged file itself, then it is included in file list, irrespective of TESTSTATUS.

Seevcover parallelmerge for more information on these and all other available options.
