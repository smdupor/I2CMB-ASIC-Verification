Power Aware Reports

During optimization, the tool generates various Power Aware reports (at the pa_reports/ directory in the current working directory) depending upon the values that you provide to the vopt -pa_genrpt argument.
Table 1. Power Aware Reports

Report

Description

vopt Argument

Architecture Report (report.pa.txt)

Contains information related to the Power Aware architecture that results from the power intent defined in the UPF file, when applied to the design.

 
-pa_genrpt=pa[+b]
Note:

Specify the argument value “b” with “pa” to include bitwise expanded information in the report.

Ack Port Driver Report (report.ack.driver.txt)

Contains information related to the acknowledge (ack) port drivers of the power switch cells.

None

Note:

The report is generated only when there is at least one power switch with an ack port in your design. If you do not specify the HDL driver of the ack port, the tool does not report the ack port. In this case, the ack port is driven by the logic specified in the IEEE Std 1801.

Connection Report (report.connection.txt)

Contains information related to the connections between your UPF, HDL, and Liberty files.

 
-pa_genrpt=conn

Design Element Report (report.de.txt)

Contains information related to the elements present in your design, and the corresponding Power Aware information.

 
-pa_genrpt=de[+b]
Note:

Specify the argument value “b” with “pa” to include bitwise expanded information in the report.

Macro Cell Report (report.cell.txt)

Contains information related to the macro cells present in your Power Aware design and the connectivity of the liberty cell related to the macro cell, if any.

 
-pa_genrpt=cell

Missing Liberty Cell Report (report.missingliberty.txt)

Contains a list of cells that do not have any Liberty cell definition.

 
-pa_libertyfiles=
<liberty_filename> | [-pa_loadlibertydb=
<database_pathname>]

Multi-Rail Cell Report (report.multiRail.cells.txt)

Contains a list of multi-rail cells that do not have any explicit UPF connections.

None

Note:

The report is generated only when the design has at least one multi-rail cell that does not have any explicit UPF connection.

Non-Retention Synchronous Flop Report (report.nretsyncff.txt)

Contains a list of hierarchical paths of flip-flops that are non-retention in nature and do not have an asynchronous control signal.

 
-pa_checks=npu
Note:

The report is generated only when the design has at least one flip-flop that is non-retention in nature, and does not have an asynchronous control signal.

Power Cell Report (report.powercells.txt)

Contains a list of unconnected cells that the simulator connects to the always-on power supply. These cells are either present inside a macro cell, or are level shifter cells.

 
[-pa_enable=
conninsidemacro] | 
[-pa_enable=
autoconnls]
Note:

The report is generated only when there is at least one unconnected cell present inside a macro cell or an unconnected level shifter cell.

PST Analysis Report (report.pst.txt)

Contains information related to the PSTs in your design, including the composition details.

The report contains tags (strings within brackets), which also appear in the static check report. 

-pa_genrpt=pst

Source Sink Path Report (report.srcsink.txt)

Contains information related to the source sink-paths that are analyzed for static checks.

See “Path-ID Link”.

 
-pa_genrpt=srcsink

Static Check Report (report.static.txt)

Contains information related to the static checks performed on your design.

A database file (report.static.tdb) is also generated, which enables you to use the Results Analysis window to view the static check results.

See “Path-ID Link”.

Restriction:

The database file is not available for ModelSim SE.

 
-pa_checks=s

Supply Network Initialization Report (report.supplynetworkinit.txt)

Contains initial value of the UPF input supply port, output supply port (if the port has no driver), default isolation and retention supply set of a power domain, and switch supply set of a power domain. These initial values are set by the tool.

 
-pa_genrpt=
supplynetworkinit

Unassociated Cell Report (report.unassociated.cells.txt)

Contains a list of power management cells that are not associated with any UPF strategy.

None

Note:

The report is generated only when there is at least one power management cell that is not associated with any UPF strategy.
