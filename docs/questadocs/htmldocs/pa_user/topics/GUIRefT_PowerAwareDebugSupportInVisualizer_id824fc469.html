Power Aware Debug Support in Visualizer

Visualizer has various debug features, and most of the features are available by default.

Objects

Window

Debug Feature

vopt option

Connectivity Tracer

  • Highlights all Power Aware objects in the window with each power domain in a different color.

  • Displays the Power Aware information when you hover over the Power Aware object.

Available by default

Design

  • Specifies the power domain of a design object in brackets.

  • Displays the Power Aware objects such as isolation, level shifter, and repeater cells.

  • Displays the following information when you hover over the Power Aware object:

    • Power domain

    • Current simstate

    • UPF file where you define the power domain

    • Simstate behavior, if you specify it using the set_simstate_behavior command

    • UPF or Liberty attribute notification, if any

  • If the module contains Liberty, Visualizer displays the following icons in front of the module:

    • — Module contains Liberty information

    • — Module is defined as a cell using `celldefine attribute

    • — Module is defined as a cell using `celldefine attribute and contains Liberty information

Available by default

Logic Cone

  • Highlights all Power Aware objects in the window with each power domain in a different color.

  • Displays the Power Aware information when you hover over the Power Aware object.

Available by default

Time Cone

  • Highlights all Power Aware objects in the window with each power domain in a different color.

  • Displays the Power Aware information when you hover over the Power Aware object.

Available by default

PA Crossings

  • Displays information about the power domain crossings.

  • Displays the static RTL checks performed during the execution of the vopt command.

‑pa_checks=s

PA Domains

Displays information about the power domains that you define in the UPF files.

Available by default

PA SimChecks

Displays information about all Power Aware dynamic checks that are performed during the Power Aware simulation.

‑pa_checks=d

PA Liberty

Displays information about the Liberty cell.

Available by default

PA States

Displays information about the PST and power states that you define in the UPF files.

Available by default

PA Supply Network

Displays a schematic view of the UPF supply network you specify in the UPF files.

Available by default

Schematic

  • Highlights all Power Aware objects in the window with each power domain in a different color.

  • Displays the Power Aware information when you hover over the Power Aware object.

Available by default

Source

  • The title bar displays the power domain and simstate of the module. The title bar also shows the simstate behavior if you specify it using the set_simstate_behavior command.

  • Highlights Power Aware information with different colors.

  • Displays the following information when you hover over a Power Aware object:

    • Details of Power Aware cells, such as isolation, and level shifter.

    • Attributes of a signal that you specify with the set_port_attributes command.

Available by default

Variables

  • The Name column has a PA-Info section, which displays the Power Aware variables.

  • Displays the Power Aware annotations, such as level shifter, and isolation, applied on the variable.

  • Displays power domains, supply sets, supply nets, and ports.

  • Driver and receiver tracing of the following UPF objects:

    • Supply Set

    • Supply Net

    • Supply Port

    • Logic Net

    • Logic Port

Available by default

Wave

  • Displays current simstate, module name of the object if the object is originating from a `celldefine module, and name of the liberty file, if any, when you hover over a Power Aware object.

  • The PA Activity Wave menu item displays all domain activity in the entire design.

  • Highlights isolation, corruption, and retention.

Available by default
