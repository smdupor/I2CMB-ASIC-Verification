Dynamic Retention Checks

Dynamic retention checks report any inconsistent retention condition in your RTL and gate-level designs.

To enable dynamic retention checks, specify a value to the vopt -pa_checks command as shown in the following table. All dynamic retention check results are written to the Transcript window.

Table 1. Dynamic Retention Checks

vopt Argument

Mnemonic

Description

-pa_checks=rcs

QPA_RET_CLK_STATE

Flags violation if the clock or latch enable is not at a certain value when the save and restore events take place. If a latch is enabled and can change its value, triggering retention can potentially cause race conditions in the stored value. This is also a check against such conditions.

-pa_checks=rop

QPA_RET_OFF_PSO

Master-Slave Configuration — Flags violation if the retention condition is not asserted when the power is switched off.

Balloon-Latch Configuration — Flags violation if the retention condition is not asserted when the power is switched off.

This check is not activated if the asynchronous set or reset is active.

-pa_checks=rpo

QPA_RET_PD_OFF

Master-Slave Configuration — Flags violation if there is an error in the sequence of triggering of the retention condition and power signal. For retention to succeed, the retention condition should be high at power-down and power-up. However, this check is triggered when that is not the case.

Balloon-Latch Configuration — Flags violation if there is an error in the sequence of triggering of the retention condition and power signal. For retention to succeed, the power should be high. However, this check is triggered when that is not the case.

This check is not activated if the asynchronous set or reset is active.

-pa_checks=rsa

QPA_RET_SEQ_ACT

Flags violation if the clock toggles during the retention period.

Note:

This check is for balloon-latch configuration only; it does not apply to master-slave (slave-alive) retention.

-pa_checks=rtcon

QPA_RET_ON_RTC

Flags violation if the retention condition is off when the power is enabled.

-pa_checks=rtcoff

QPA_RET_OFF_RTC

Flags violation if the retention condition is off when the power is disabled.

-pa_checks=rtctog

QPA_RET_RTC_TOGGLE

Flags violation if the retention condition toggles during power-down. Toggling of the retention condition during power-down corrupts the retained value.

-pa_checks=rtc

  

Enables the following retention condition checks:

  • -pa_checks=rtcon

  • -pa_checks=rtcoff

  • -pa_checks=rtctog

-pa_checks=r

  

Enables all dynamic retention checks (rcs, rop, rpo, rsa, rtcon, rtcoff, and rtctog).
