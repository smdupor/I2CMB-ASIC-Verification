Parameters are VRM variables. They are either VRM pre-defined parameters or user-defined. The parameter value can reference another parameter name that was defined previously. User-defined parameters can have the types:

  1. text: the normal parameter value
  2. file: the value refer to a file
  3. TCL: the parameter value is TCL expanded
Exporting a user-defined parameter allows scripts to use it as an Environment Variable Among the mostly used pre-defined parameters are:
  • (%DATADIR%):- Directory path of VRMDATA directory
  • (%RMDBDIR%):- Directory path of RMDB file
  • (%VRUNDIR%):- Directory path from where VRM is invoked
  • (%TASKDIR%):- Working directory in which the current task is being executed

Parameters-expanded fields are fields that allow referencing parameters. These fields has auto-complete feature that allows you to select from the set of existing parameters. In order to reference a parameter, you can use, "(%parametername%)". If the field is parameter-expanded, once you write "(%" you will get a list of parameters you can select from. Also, in the help links attached with every field, you can find a note stating that the field is parameter-expanded.
